# 📚 AI Study Assistant

**An intelligent study companion powered by Google Gemini AI**

A Flutter mobile application that acts as an AI-powered study assistant for students. Ask questions, share images, upload documents, and receive intelligent academic support — all synced across devices via Firebase.

## ✨ Features

| | Feature | Description |
|---|---|---|
| 🤖 | **AI Conversations** | Chat with Gemini 2.5 Flash for academic help in Turkish or English |
| 📷 | **Image Analysis** | Share photos from camera or gallery — Gemini reads and analyzes them |
| 📄 | **PDF Support** | Upload PDFs and Gemini will analyze the actual content |
| 💬 | **Cross-Device History** | All chats stored in Firestore, accessible from any device |
| ☁️ | **Cloud File Storage** | Files uploaded to S3-compatible MinIO — never stored locally |
| 🔍 | **Chat Search** | Search through past conversations from the side drawer |
| 🌙 | **Dark / Light Theme** | Toggle and persists across sessions |
| 📶 | **Offline Detection** | Real-time connectivity banner; sending blocked when offline |
| 🔒 | **Secure Auth** | Google Sign-In & Email/Password via Firebase Authentication |
| 👤 | **Profile Editing** | Update display name, email, and password with re-authentication |
| 🖼️ | **Fullscreen Viewer** | Tap images to view fullscreen with pinch-to-zoom; long-press to save |

## 🚀 Getting Started

### Prerequisites

- Flutter SDK `^3.10.1`
- A Firebase project (Auth + Firestore enabled)
- A [Google Gemini API key](https://aistudio.google.com/app/apikey)
- A MinIO instance or any S3-compatible object storage

### 1 — Clone & install

```bash
git clone https://github.com/bmuftuoglu/student-help-desk.git
cd student-help-desk
flutter pub get
```

### 2 — Firebase setup

1. Create a project at [console.firebase.google.com](https://console.firebase.google.com)
2. Enable **Authentication** → Google Sign-In + Email/Password
3. Enable **Cloud Firestore**
4. Download `google-services.json` and place it in `android/app/`
5. Place `lib/firebase_options.dart` (generated by `flutterfire configure`)

### 3 — Create `lib/secrets.dart`

> ⚠️ This file is in `.gitignore` — **never commit it.**

```dart
// lib/secrets.dart
const String geminiApiKey = 'YOUR_GEMINI_API_KEY';

const String s3Endpoint  = 'your-minio-host.com'; // no https://
const String s3Bucket    = 'your-bucket';
const String s3AccessKey = 'your-access-key';
const String s3SecretKey = 'your-secret-key';
const String s3Region    = 'us-east-1';
```

### 4 — Run

```bash
flutter run
```

## 🔐 Security Notes

- `lib/secrets.dart` is listed in `.gitignore` — API keys are never committed
- Firestore security rules should restrict reads/writes to authenticated users
- MinIO bucket is configured for public GET access (read-only); writes require credentials signed with AWS Signature V4

## 📋 Firestore Data Model

```
users/{uid}/
├── sessions/{sessionId}
│   ├── title: string
│   ├── createdAt: timestamp
│   ├── updatedAt: timestamp
│   ├── lastUserPrompt: string
│   └── lastAiResponse: string
│   └── messages/{messageId}
│       ├── role: "user" | "assistant"
│       ├── content: string
│       ├── fileUrl: string?       ← MinIO URL
│       ├── fileName: string?
│       ├── mimeType: string?
│       └── createdAt: timestamp
└── questionHistory/{id}
    ├── prompt: string
    ├── sessionId: string
    └── createdAt: timestamp
```
